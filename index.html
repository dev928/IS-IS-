IS-IS (Intermediate System to Intermediate System) Protocol

A comprehensive overview of the routing protocol used in large-scale networks and service provider environments.

1. Overview of IS-IS Protocol

  • Link-state routing protocol operating within a single administrative domain
  • Classified as an Interior Gateway Protocol (IGP), similar to OSPF
  • Widely used in large-scale networks, service provider environments, and ISP backbones
  • Runs directly over Layer 2 (Data Link Layer)
  • Supports both IPv4 and IPv6 through multi-topology IS-IS

2. Characteristics of IS-IS

Feature Description
Protocol Type Link-State Routing Protocol
Algorithm Dijkstra's Shortest Path First (SPF)
Metric Type Cost-based (default value of 10 per interface)
Transport Mechanism Operates directly over Layer 2 (no dependency on IP)
Hierarchical Structure Supports two levels (Level-1 and Level-2)
Network Type Supports IPv4, IPv6, and MPLS
Scalability Used in large ISPs and enterprise networks

3. IS-IS Packet Types

Packet Type Description
ES-IS Hello (ESH) Used between an End System (ES) and an Intermediate System (IS) for discovery
IS-IS Hello (IIH) Used between IS-IS routers to establish and maintain adjacencies
Link State PDU (LSP) Advertises routing information about connected routers and networks
Sequence Number PDU (SNP) Ensures database synchronization between routers
Complete Sequence Number PDU (CSNP) Sends a full list of LSPs to ensure consistency

4. IS-IS Hierarchical Structure

Level Function
Level 1 (L1) Intra-area routing (similar to OSPF intra-area)
Level 2 (L2) Inter-area routing (similar to OSPF backbone)
Level 1-2 (L1/L2) Routers that act as both L1 and L2, connecting areas
  • Level 1 routers: Can only see routes inside their area
  • Level 2 routers: Maintain inter-area routing
  • Level 1-2 routers: Act as a bridge between Level 1 and Level 2 networks

5. IS-IS Network Types

5.1 Point-to-Point Networks

  • Two routers directly connected
  • Both routers exchange Hello packets and establish an adjacency

5.2 Broadcast (Multi-Access) Networks

  • No DR (Designated Router) like OSPF
  • Elects a Designated Intermediate System (DIS) to manage LSP flooding

6. IS-IS vs OSPF Comparison

Feature IS-IS OSPF
Protocol Origin OSI Model IP Model
Encapsulation Layer 2 (Ethernet) IP (Protocol 89)
Area Types Level 1, Level 2, L1/L2 Backbone (0), Normal, Stub, NSSA
Metric Calculation Simple cost (default 10) Bandwidth-based cost
Scalability More scalable for large ISPs Best for enterprise networks
IPv6 Support Uses Multi-Topology IS-IS Uses OSPFv3

7. IS-IS Configuration on Cisco Routers

# Step 1: Enable IS-IS Process
router isis
 net 49.0001.0000.0000.0001.00  # IS-IS NET Address

# Step 2: Configure Interfaces
interface GigabitEthernet0/0
 ip router isis
 isis network point-to-point

# Step 3: Define IS-IS Levels
router isis
 is-type level-2

# Step 4: Enable IPv6 Routing
router isis
 address-family ipv6

8. Advantages and Disadvantages

Advantages

  • Highly scalable (used in large ISP networks)
  • Less overhead since it runs over Layer 2
  • Supports IPv4, IPv6, and MPLS without major changes
  • Faster convergence compared to OSPF in large networks

Disadvantages

  • More complex configuration compared to OSPF
  • Not as widely adopted in enterprises (mainly used in ISPs)
  • Uses CLNS addressing, which is not as intuitive as IP addressing

9. Conclusion

IS-IS is a powerful, scalable, and flexible link-state routing protocol used in large ISP and service provider networks. It offers:

  • Better scalability due to simple metric system
  • Efficient LSP flooding
  • Preferred choice for ISPs and large-scale networking deployments
